## Shared Implementation Notes

### Root div pattern (every template)

```tsx
<div
  className="w-full h-full bg-background flex flex-col overflow-hidden"
  style={{ padding: '60px 80px' }}
>
  {header && <div className="flex-shrink-0 mb-10">{header}</div>}
  {/* content â€” typically a flex-1 container */}
</div>
```

Full-bleed slides (no padding, fills entire frame) swap this for:

```tsx
<div className="w-full h-full bg-background relative overflow-hidden flex items-center justify-center">
  <TitleChrome />
  {/* centered content */}
</div>
```

### Tailwind token classes (use only these â€” no hardcoded hex values)

| Token | Class |
|---|---|
| Background | `bg-background` / `text-background` |
| Surface | `bg-surface` / `border-surface` |
| Accent | `bg-accent` / `text-accent` |
| Text | `text-slide-text` |
| Muted | `text-muted` |
| Fonts | `font-display` / `font-body` / `font-mono` |

When an inline `style` is unavoidable, use CSS variables: `rgb(var(--color-accent))`, `var(--font-display)`.

### Existing keyframes to reuse (no new ones needed for this batch)

| Keyframe class | What it does | Where to use |
|---|---|---|
| `animate-fade-up` | Opacity 0â†’1 + translateY 12pxâ†’0, 600ms | Any staggered content reveal |
| `animate-letter-rise` | Character springs up from below, 700ms | Full-bleed headline characters |
| `animate-line-expand` | Horizontal bar scales from center, 400ms | Decorative accent lines |
| `animate-subtitle-rise` | Subtitle fades up, 600ms | Subtitles appearing after titles |
| `animate-accent-bar` | Accent bar scales from left, 350ms | Left-edge or underline bars |
| `animate-hammer-drop` | Element drops from above with bounce, 700ms | A single dominant hero element |

Apply stagger with `animationDelay`:

```tsx
style={{ animationDelay: `${i * 50}ms` }}
```

### Export mode handling

Import `isExportMode` from `../../src/utils/export` in any template that uses looping or staged `setTimeout`/`setInterval` animations. One-shot animations (run once, land, done) need no guard. Staged step-by-step reveals (like the stagger loops below) need a guard to pre-show everything in export mode:

```tsx
import { isExportMode } from '../../utils/export'

// In useEffect â€” skip animation, show final state immediately
const [visibleCount, setVisibleCount] = useState(isExportMode ? items.length : 0)
useEffect(() => {
  if (isExportMode) return
  // ... stagger logic
}, [visibleCount])
```

### `QuoteSlide`

**Directory:** `src/templates/quote/`

**When to use:** A single large pull quote with attribution â€” high visual impact, zero clutter. Use for memorable statements, framing slides, or chapter openers.

**Props:**

```ts
export interface QuoteSlideProps {
  quote: string          // The quote text (keep under ~120 chars for best rendering)
  attribution?: string   // Person's name
  role?: string          // Title or role of the speaker
  company?: string       // Company or context
}
```

**Layout:** No `header` prop â€” the quote is the entire slide. Full-bleed with content vertically and horizontally centered.

- Large decorative opening `"` character: `font-display`, ~160px, `text-accent`, absolute position top-left of the content block, `opacity: 0.15`.
- Quote text: `font-display`, 48â€“56px (use `clamp(38px, 4vw, 56px)` inline), `text-slide-text`, `lineHeight: 1.25`, max-width ~800px, centered.
- Attribution line: `font-body`, 18px, `text-muted`. Prefixed with an em dash (`â€”`). `role` and `company` separated by `Â·` if both present.
- Thin `w-10 h-0.5 bg-accent` accent bar above the attribution line, centered.
- No `TitleChrome` â€” the quote itself is the visual focus.

**Animation:** Quote fades in on mount (`animate-fade-up` with `animationFillMode: 'both'`). Attribution and bar `animate-fade-up` with `animationDelay: '300ms'`.

### `SectionDividerSlide`

**Directory:** `src/templates/sectiondivider/`

**When to use:** A full-bleed, visually bold static section break between deck chapters. No animation â€” the high-contrast layout does the work. Distinct from `SectionTitleSlide` (which has split-flap animation); use `SectionDividerSlide` when a clean, immediate reset is preferred.

**Props:**

**Layout:** Direct string props â€” same contract as `SectionTitleSlide`. No `header` or `children`.

**Animation:** None â€” this is an instant, static transition.

### `ClosingSlide`

**Directory:** `src/templates/closing/`

**When to use:** Final "Thank You" slide with a call to action and/or contact info. Mirrors `TitleSlide` energy. Use instead of (or in addition to) `TheEndSlide` when you need a URL, email, or social handle on the final slide.

**Props:**

```ts

  headline?: string     // Default: "Thank You"
  subtitle?: string     // One-line CTA or closing thought
  contact?: string      // Email, URL, or @handle
  children?: ReactNode  // Optional: QR code, logo, or extra content
}
```

**Layout:** Full-bleed, centered â€” same shell as `TheEndSlide`.

- Uses `TitleChrome` for visual consistency with the deck opener.
- Headline: `font-display`, bold, 80px, `text-slide-text`, `lineHeight: 1` â€” rendered character-by-character with staggered `animate-letter-rise` (same technique as `TheEndSlide`). Default: `"Thank You"`.
- Accent line: `w-16 h-0.5 bg-accent animate-line-expand` with `animationDelay` timed to appear after the last letter (same calculation as `TheEndSlide`'s `lineDelay`).
- Subtitle: `font-body`, 22px, `text-muted`, `animate-subtitle-rise` with further delay.
- Contact line: `font-mono`, 14px, `text-accent`, `0.1em` letter-spacing. Lucide `Globe` icon (URLs), `Mail` (email), or `AtSign` (@handles) to the left, `size={16}`. Appears after subtitle with additional 200ms delay.
- `children` renders below contact with `marginTop: '32px'`.

**Animation:** Identical approach to `TheEndSlide` â€” `letter-rise` per character, then `line-expand`, then `subtitle-rise`. All animations are one-shot; no `isExportMode` guard needed.

### `AgendaSlide`

**Directory:** `src/templates/agenda/`

**When to use:** A numbered list of topics at the start of a deck with optional time estimates. More structured than `SplitFlapBulletSlide` (use that for a dramatic reveal); use this for a clean, scannable agenda.

**Props:**

```ts

  title: string
  duration?: string   // e.g. "10 min", "~15 min"
}

export interface AgendaSlideProps {
  header?: ReactNode
  items: AgendaItem[]
}
```

**Layout:** Standard padded content slide.

- `header` prop supported (typically `<SectionTitle title="Agenda" />`).
- Items rendered as a vertical stack. Each item is a horizontal flex row:
  - Left column (min-width ~64px): sequential number in `font-mono`, `text-accent`, ~44px, right-aligned.
  - Center: `title` in `font-display`, 28px, `text-slide-text`, `flex-1`.
  - Right (if `duration`): `duration` string in `font-mono`, 14px, `text-muted`, right-aligned.
  - Full-width `border-b border-surface` divider below each row except the last.
- Row height: `py-5`. Cap at ~6 items (document in JSDoc; don't enforce in code).

**Animation:** Items stagger in with `animate-fade-up` at 50ms per item. In export mode, show all items immediately:

```tsx
const [visible, setVisible] = useState(isExportMode ? items.length : 0)
useEffect(() => {
  if (isExportMode || visible >= items.length) return
  const id = setTimeout(() => setVisible(v => v + 1), 50)
  return () => clearTimeout(id)
}, [visible, items.length])
```

Render items conditionally: `items.slice(0, visible)`.

## PR 5 â€” People

### Avatar fallback helper (shared by both templates)

Define a small helper in each template file (or extract to `src/templates/common/Avatar.tsx` if it feels cleaner):

```tsx
function getInitials(name: string): string {
  return name.split(' ').map(p => p[0]).slice(0, 2).join('').toUpperCase()
}

function Avatar({ src, name, size = 48 }: { src?: string; name: string; size?: number }) {
  if (src) return <img src={src} alt={name} className="rounded-full object-cover" style={{ width: size, height: size }} />
  return (
```
