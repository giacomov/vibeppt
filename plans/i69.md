# Implementation Plan — New Template Library (Issue #69)

This plan adds **15 new templates** to VibePPT's template library. Existing templates not listed here (`TitleSlide`, `ChartSlide`, `TheEndSlide`, `SectionTitleSlide`, etc.) remain unchanged. Work is split into five focused PRs that each ship a coherent theme cluster.

---

## Background: How Templates Work

All templates follow the same structural contract. Study these files before implementing anything:

| File | Why read it |
|---|---|
| `src/theme/tokens.ts` | Source of truth for all design tokens |
| `src/templates/common/SlideTitle.tsx` | `HeroTitle`, `SectionTitle`, `SubsectionTitle` — used in every `header` prop |
| `src/templates/common/TitleChrome.tsx` | Decorative background layer used in full-bleed slides |
| `tailwind.config.ts` | Token → Tailwind class mapping and animation keyframes |
| `src/templates/cards/CardSlide.tsx` | Reference for animation patterns |
| `src/templates/keytakeaway/KeyTakeawaySlide.tsx` | Reference for stateful click-to-reveal |
| `src/templates/title/TitleSlide.tsx` | Reference for full-bleed, chrome-decorated slides |
| `src/types/slide.ts` | `SlideComponent`, `SlideMeta`, `Deck` types |

### Structural rules (every template must follow these)

```tsx
// Root div: always these exact classes + padding
<div
  className="w-full h-full bg-background flex flex-col overflow-hidden"
  style={{ padding: '60px 80px' }}
>
  {header && <div className="flex-shrink-0 mb-10">{header}</div>}
  {/* content — typically flex-1 */}
</div>
```

- **Tailwind token classes only:** `bg-background`, `bg-surface`, `bg-accent`, `text-slide-text`, `text-muted`, `text-accent`, `font-display`, `font-body`, `font-mono`, `border-surface`. Never hardcode hex values.
- **CSS variable fallback** for situations where inline style is unavoidable: `rgb(var(--color-accent))`, `var(--font-display)`.
- **No magic strings.** Font names come from CSS variables (`var(--font-display)` etc.) so deck-level theme overrides work.
- Use Lucide React (`size={22}`) for icons. Never pass raw emoji strings except in `PrismSlide`'s explicitly dual-mode icon fields.
- Each template directory: `src/templates/<name>/` with `<Name>Slide.tsx` + `example.tsx`. Add a one-liner JSDoc comment at the top of each file stating **when to use it**.

### After every PR

1. `npm run build` — fix all TypeScript errors.
2. `npm run export -- --deck=<demo-deck> --format=png` — visually audit every exported PNG.
3. Update `CLAUDE.md` template catalog table and template decision guide with the new entries.

## PR 1 — Deck Bookmarks

**Templates:** `QuoteSlide`, `SectionDividerSlide`, `ClosingSlide`, `AgendaSlide`

These four templates bookend and structure a deck: dramatic openers, section resets, the closing call to action, and the agenda roadmap. They are the highest-priority gap in the current library.

---

### `QuoteSlide`

**Directory:** `src/templates/quote/`

**When to use:** A single large pull quote with attribution — high visual impact, zero clutter. Use for memorable statements, framing slides, or chapter openers.

**Props:**

```ts
export interface QuoteSlideProps {
  quote: string          // The quote text (keep under ~120 chars for best rendering)
  attribution?: string   // Person's name
  role?: string          // Title or role of the speaker
  company?: string       // Company or context
}
```

**Layout:** No `header` prop — the quote is the entire slide. Vertically and horizontally center the content.

- Large decorative opening `"` character: `font-display`, ~160px, `text-accent`, `opacity-20`, absolutely positioned top-left.
- Quote text: `font-display`, 48–56px, `text-slide-text`, `lineHeight: 1.25`, max-width ~800px, centered.
- Attribution line: `font-body`, 18px, `text-muted`. Prefixed with an em dash (`—`). `role` and `company` separated by `·` if present.
- Thin `w-10 h-0.5 bg-accent` accent bar above the attribution line.
- No `TitleChrome` — the quote itself is the visual focus.

**Animation:** Fade the quote in (`opacity 0 → 1`, `translateY 8px → 0`, 600ms ease-out) on mount.

---

### `SectionDividerSlide`

**Directory:** `src/templates/sectiondivider/`

**When to use:** Full-bleed, visually bold section break. No animation — instant, high-contrast reset between deck chapters. Distinct from `SectionTitleSlide` (which has split-flap animation); use this when you want a clean, static divider.

**Props:**

```ts
export interface SectionDividerSlideProps {
  title: string
  eyebrow?: string    // e.g. "Part 2", "Chapter 3"
  subtitle?: string
}
```

**Layout:** Direct string props (no `header` prop, no `children`) — same contract as `SectionTitleSlide`.

- Uses `TitleChrome` for the dot-grid and corner watermark decorations.
- `bg-background` base with a large accent-colored vertical bar on the left edge (8px wide, full height, `bg-accent`).
- Eyebrow: `font-mono`, 12px, `text-accent`, `0.28em` letter-spacing, uppercase.
- Title: `font-display`, 72–80px, `text-slide-text`, `lineHeight: 1.05`.
- Subtitle: `font-body`, 20px, `text-muted`.
- Content block: vertically centered on the slide (use `justify-center` on the flex column).

**Animation:** None. This is a static slide — the contrast does the work.

---

### `ClosingSlide`

**Directory:** `src/templates/closing/`

**When to use:** Final "Thank You" slide with a call to action and/or contact info. Mirrors `TitleSlide` energy. Distinct from `TheEndSlide` (which is minimal); use this when you need a CTA, website, email, or social handle.

**Props:**

```ts
export interface ClosingSlideProps {
  headline?: string     // Default: "Thank You"
  subtitle?: string     // One-line CTA or closing thought
  contact?: string      // Email, URL, or @handle
  children?: ReactNode  // Optional: anything extra (QR code, logo, etc.)
}
```

**Layout:**

- Uses `TitleChrome` (same as `TitleSlide` and `TheEndSlide`) for visual consistency.
- `HeroTitle` is **not** used here — the headline renders at 80px `font-display` directly (slightly smaller than the full hero to give room to contact info).
- Contact line: `font-mono`, 14px, `text-accent`, 0.12em letter-spacing, with a small `Link` or `Globe` Lucide icon to the left.
- Thin `w-16 h-px bg-surface` separator between subtitle and contact.
- `children` rendered below the contact line (for QR codes, logos, etc.).
- Layout is vertically centered with `justify-center`.

**Animation:** Same subtle entrance as `TitleSlide` — `subtitle-rise` on subtitle, `accent-bar` on the decorative line.

---

### `AgendaSlide`

**Directory:** `src/templates/agenda/`

**When to use:** Numbered list of topics at the start of a deck — the roadmap slide. Use `SplitFlapBulletSlide` for agenda items with a dramatic reveal; use this for a clean, scannable agenda with optional timing.

**Props:**

```ts
export interface AgendaItem {
  title: string
  duration?: string   // e.g. "10 min", "~15 min"
}

export interface AgendaSlideProps {
  header?: ReactNode
  items: AgendaItem[]
}
```

**Layout:**

- `header` prop supported (typically `<SectionTitle title="Agenda" />`).
- Items rendered as a vertical stack. Each item is a horizontal row:
  - Left: large sequential number in `font-mono`, `text-accent`, ~48px, min-width ~64px. Right-aligned within its column.
  - Center: `title` in `font-display`, 28px, `text-slide-text`. If `duration` is present, it renders inline to the right of the title in `font-mono`, 14px, `text-muted`.
  - Full-width `border-b border-surface` divider below each row except the last.
- Row height: `py-5` (uniform). Max ~6 items before text gets too small — the template doesn't enforce this, but `example.tsx` should show 5.
- Items animate in with staggered `fade-up` (50ms per item).

## PR 2 — Information Architecture

**Templates:** `TableOfContentsSlide`, `IconGridSlide`, `MatrixSlide`

These three templates display structured information spatially — section overviews, feature grids, and strategic quadrant maps.

---

### `TableOfContentsSlide`

**Directory:** `src/templates/toc/`

**When to use:** A richer visual overview of deck sections than `AgendaSlide`. Shows section names with icons or numbers in a grid or multi-column layout. Use when sections benefit from visual hierarchy beyond a simple numbered list.

**Props:**

```ts
export interface TocSection {
  title: string
  eyebrow?: string      // e.g. section number or tag
  icon?: ReactNode      // Lucide icon
  description?: string  // One-line blurb
}

export interface TableOfContentsSlideProps {
  header?: ReactNode
  sections: TocSection[]
}
```

**Layout:**

- `header` prop supported.
- Responsive column grid: 2 columns for ≤4 items, 3 columns for 5–6, 4 columns for 7–8. Implemented with CSS grid (`grid-cols-2`, `grid-cols-3`, `grid-cols-4`).
- Each section card: `bg-surface` rounded box, `p-6`. Icon top-left in `text-accent`. Eyebrow below icon in `font-mono` 11px `text-muted`. Title in `font-display` 22px. Description in `font-body` 14px `text-muted`.
- Cards animate in with staggered `fade-up` (40ms per card).

---

### `IconGridSlide`

**Directory:** `src/templates/icongrid/`

**When to use:** A grid of icons with short labels and optional descriptions. Use for feature lists, product attributes, team values, or any set of 4–12 discrete items where the icon communicates faster than text.

**Props:**

```ts
export interface IconGridItem {
  icon: ReactNode      // Lucide icon, size={28}
  label: string        // Short label (1–4 words)
  description?: string // Optional one-liner
}

export interface IconGridSlideProps {
  header?: ReactNode
  items: IconGridItem[]
  columns?: 2 | 3 | 4  // Default: auto based on item count
}
```

**Layout:**

- `header` prop supported.
- CSS grid layout. Default column count: 2 for ≤4 items, 3 for 5–9, 4 for 10–12.
- Each cell: icon centered horizontally in `text-accent`, label below in `font-display` 20px centered `text-slide-text`, description in `font-body` 14px `text-muted` centered.
- Thin `bg-accent` horizontal bar (2px, width ~24px) between icon and label as a visual connector.
- Items animate in with staggered `fade-up`.

---

### `MatrixSlide`

**Directory:** `src/templates/matrix/`

**When to use:** A 2×2 strategic quadrant matrix (Boston Consulting Group matrix, Eisenhower matrix, etc.). Use when content naturally falls into four categories defined by two axes.

**Props:**

```ts
export interface MatrixQuadrant {
  title: string
  items?: string[]      // Bullet points or examples within the quadrant
  highlight?: boolean   // Renders with accent border to draw attention
}

export interface MatrixSlideProps {
  header?: ReactNode
  xAxis: { label: string; from: string; to: string }
  yAxis: { label: string; from: string; to: string }
  topLeft: MatrixQuadrant
  topRight: MatrixQuadrant
  bottomLeft: MatrixQuadrant
  bottomRight: MatrixQuadrant
}
```

**Layout:**

- `header` prop supported.
- The matrix is a CSS grid `grid-cols-2 grid-rows-2` filling available height. Each quadrant: `bg-surface` with `border border-surface`. Highlighted quadrant gets `border-accent`.
- Quadrant title: `font-display` 22px `text-slide-text`. Items: `font-body` 14px `text-muted`, rendered as a tight bullet list.
- X-axis label spans the full width below the grid, centered. Y-axis label is rotated 90° and spans the full height to the left of the grid.
- Axis from/to labels appear at the ends of each axis arrow line in `font-mono` 11px `text-accent`.
- No animation — the content reads at a glance.

## PR 3 — Time & Progression

**Templates:** `TimelineSlide`, `RoadmapSlide`, `ProcessSlide`

These three templates show sequences: past events, future milestones, and step-by-step processes. They share a common horizontal or vertical spine metaphor.

---

### `TimelineSlide`

**Directory:** `src/templates/timeline/`

**When to use:** Chronological events — history, milestones, case study narrative. Use horizontal orientation for ≤6 events, vertical for longer lists.

**Props:**

```ts
export interface TimelineEvent {
  date: string           // Display string: "Q1 2024", "March", "2019", etc.
  title: string
  description?: string
  highlight?: boolean    // Accent color for key events
}

export interface TimelineSlideProps {
  header?: ReactNode
  events: TimelineEvent[]
  orientation?: 'horizontal' | 'vertical'  // Default: 'horizontal'
}
```

**Layout (horizontal):**

- A horizontal line (`h-px bg-surface`) spans the width of the content area at vertical midpoint.
- Each event is positioned along the line. Events alternate above/below the line (odd index above, even below) to prevent crowding.
- Connector: `w-px h-6 bg-accent` vertical line from the spine to a `w-3 h-3 rounded-full bg-accent` dot. Highlighted events use a larger dot (`w-4 h-4`) with a faint accent ring.
- Date: `font-mono` 11px `text-accent` above the dot. Title: `font-display` 18px below the dot. Description: `font-body` 13px `text-muted`.

**Layout (vertical):**

- Vertical spine on the left. Events stack down the right side with connector dots.
- Same styling as horizontal but transposed.

**Animation:** Events stagger in left-to-right (horizontal) or top-to-bottom (vertical) with `fade-up` at 60ms intervals.

---

### `RoadmapSlide`

**Directory:** `src/templates/roadmap/`

**When to use:** Future-oriented horizontal timeline grouped into phases, quarters, or themes. Different from `TimelineSlide` (past events); use for "what's coming" planning slides.

**Props:**

```ts
export interface RoadmapPhase {
  label: string          // e.g. "Q1 2025", "Phase 1", "Now"
  items: string[]        // Deliverables or features in this phase
  status?: 'done' | 'active' | 'planned'  // Default: 'planned'
}

export interface RoadmapSlideProps {
  header?: ReactNode
  phases: RoadmapPhase[]  // 3–5 phases recommended
}
```

**Layout:**

- Horizontal swimlane layout. Each phase occupies an equal-width column separated by a subtle `border-r border-surface`.
- Phase label: `font-mono` 12px `text-accent` uppercase at the top of each column with a `w-full h-0.5 bg-accent` underline.
- Status coloring: `done` phases use muted text; `active` phase gets a subtle `bg-surface` column background and full-opacity text; `planned` phases get standard muted treatment.
- Items within each phase are rendered as a tight list in `font-body` 15px `text-slide-text` with a small `w-1 h-1 rounded-full bg-accent` bullet.
- No animation — roadmaps are reference slides read at leisure.

---

### `ProcessSlide`

**Directory:** `src/templates/process/`

**When to use:** A numbered step-by-step process — onboarding flow, methodology, how-it-works. More structured than `BulletSlide`; use when sequence matters. Prefer `CycleSlide` if the process loops.

**Props:**

```ts
export interface ProcessStep {
  title: string
  description?: string
  icon?: ReactNode    // Optional Lucide icon
}

export interface ProcessSlideProps {
  header?: ReactNode
  steps: ProcessStep[]
  orientation?: 'horizontal' | 'vertical'  // Default: 'horizontal' for ≤4 steps
}
```

**Layout (horizontal, ≤4 steps):**

- Steps are evenly spaced in a row. Between each step, a right-pointing `→` arrow in `text-muted`.
- Each step: large step number `font-mono` 13px `text-accent` at top. Optional icon centered. Title `font-display` 20px. Description `font-body` 14px `text-muted`.

**Layout (vertical):**

- Steps stack vertically. A `w-px bg-surface` spine runs down the left. Each step connects to the spine via a numbered circle (`w-8 h-8 rounded-full bg-accent text-background font-mono text-sm`).
- Content to the right of the spine: title `font-display` 22px, description `font-body` 15px `text-muted`.

**Animation:** Steps animate in sequentially with `fade-up` at 80ms intervals.

## PR 4 — Contrast & Layout

**Templates:** `TwoColumnSlide`, `ProblemSolutionSlide`, `BigNumberSlide`

These templates are built around visual contrast: side-by-side columns, problem–solution tension, and a single dominant number.

---

### `TwoColumnSlide`

**Directory:** `src/templates/twocolumn/`

**When to use:** Two columns of content with individual headers — richer and more structured than `SplitSlide` (which is a generic container). Use when each column has a named section with its own list or content.

**Props:**

```ts
export interface TwoColumnSlideProps {
  header?: ReactNode
  left: {
    title?: string
    icon?: ReactNode
    content: ReactNode    // Can be a list, paragraph, or any React content
  }
  right: {
    title?: string
    icon?: ReactNode
    content: ReactNode
  }
  divider?: boolean       // Default: true — shows a vertical accent line between columns
}
```

**Layout:**

- `header` prop supported.
- Two equal columns (`flex-1 flex-1`) separated by an optional `w-px bg-surface` vertical divider.
- Column header: icon (optional, `text-accent` size 20), then title in `font-display` 24px `text-slide-text`, then a `w-8 h-0.5 bg-accent` underbar.
- Content rendered below the header; styling is left to the caller.
- Columns have `gap-12` between them and `pt-2` on content below their header.

**Animation:** None — generic container; keep it predictable.

---

### `ProblemSolutionSlide`

**Directory:** `src/templates/problemsolution/`

**When to use:** Classic problem/solution framing — two panels, visual contrast. Use when making a before/after or challenge/answer argument. More opinionated than `CompareSlide` (which does a tug-of-war row animation); use this when the framing itself is the message.

**Props:**

```ts
export interface ProblemSolutionSlideProps {
  header?: ReactNode
  problem: {
    title?: string        // Default: "Problem"
    points: string[]
    icon?: ReactNode
  }
  solution: {
    title?: string        // Default: "Solution"
    points: string[]
    icon?: ReactNode
  }
}
```

**Layout:**

- Two panels side by side, separated by a thin `w-px bg-surface` divider.
- **Problem panel (left):** slightly warmer muted treatment — `text-muted` bullets, icon in `text-muted`. Title `font-display` 22px.
- **Solution panel (right):** accent treatment — `text-accent` icon, accent-colored bullet dots. Title `font-display` 22px.
- Each bullet: `font-body` 18px `text-slide-text` with `w-1.5 h-1.5 rounded-full` bullet dot. Problem dots `bg-muted`, solution dots `bg-accent`.
- A small `→` or transition arrow in `text-accent` centered on the divider, vertically mid-panel, signals direction.

**Animation:** Problem panel fades in first, then solution panel fades in 200ms later.

---

### `BigNumberSlide`

**Directory:** `src/templates/bignumber/`

**When to use:** A single dominant metric or statistic that speaks for itself — revenue, users, NPS score, percentage improvement. One number, maximum impact. Use before a chart slide to set up the punchline.

**Props:**

```ts
export interface BigNumberSlideProps {
  number: string          // Display string: "3.2M", "94%", "$1.2B", "+47%"
  label: string           // What it measures: "Monthly Active Users"
  context?: string        // Supporting sentence: "As of Q4 2024, up 22% YoY"
  trend?: 'up' | 'down' | 'flat'  // Optional directional indicator
}
```

**Layout:** No `header` prop — the number is the slide. Vertically and horizontally centered.

- Number: `font-display`, 120–140px (scale to fit), `text-accent`, `lineHeight: 1`.
- Trend arrow: if `trend` is set, render a Lucide `TrendingUp`, `TrendingDown`, or `Minus` icon to the right of the number, `size={48}`, color-coded: `text-accent` for up, `text-muted` for down/flat.
- Label: `font-body` 24px `text-muted`, centered below the number, with a `w-16 h-px bg-surface` separator.
- Context: `font-body` 16px `text-muted`, centered, below the label.

**Animation:** Number animates in with `hammer-drop` (existing keyframe) — it snaps down from above. Label and context `fade-up` 200ms later.

## PR 5 — People

**Templates:** `TeamSlide`, `TestimonialSlide`

Both templates center on humans: faces, names, words. They require special handling for avatar images (graceful fallback when no photo is available).

---

### `TeamSlide`

**Directory:** `src/templates/team/`

**When to use:** Introduce the team — speakers, contributors, or org structure. Each card shows a person's name, role, and optional avatar.

**Props:**

```ts
export interface TeamMember {
  name: string
  role: string
  avatar?: string         // URL to photo (optional — renders initials placeholder if absent)
  bio?: string            // One-line description
  highlight?: boolean     // Accent border for the presenter/focus person
}

export interface TeamSlideProps {
  header?: ReactNode
  members: TeamMember[]
}
```

**Layout:**

- `header` prop supported.
- Responsive grid: 2 columns for ≤4 members, 3 for 5–6, 4 for 7–8.
- Each member card: `bg-surface` rounded box.
  - **Avatar:** `w-16 h-16 rounded-full` image. If `avatar` is absent, render initials (`firstName[0] + lastName[0]`) in a `bg-accent text-background font-display` circle.
  - **Name:** `font-display` 20px `text-slide-text`.
  - **Role:** `font-mono` 11px `text-accent` uppercase, `0.1em` letter-spacing.
  - **Bio:** `font-body` 13px `text-muted`.
- Highlighted member card gets a `border border-accent` outline.

**Animation:** Cards stagger in with `fade-up` at 50ms intervals.

---

### `TestimonialSlide`

**Directory:** `src/templates/testimonial/`

**When to use:** A customer or user quote with the person's identity visible — name, role, company, and optional avatar. More social-proof-focused than `QuoteSlide` (which is purely typographic); use this when the source's credibility is part of the message.

**Props:**

```ts
export interface TestimonialSlideProps {
  quote: string          // The testimonial text
  name: string
  role?: string
  company?: string
  avatar?: string        // URL or omit for initials fallback
  rating?: 1 | 2 | 3 | 4 | 5  // Optional star rating
}
```

**Layout:** No `header` prop — the testimonial is the slide. Vertically centered.

- Large decorative opening `"` in `font-display` ~120px `text-accent opacity-20`, absolutely positioned.
- Quote text: `font-display` 36–44px `text-slide-text`, max-width ~780px, centered.
- Separator: `w-10 h-0.5 bg-accent` centered below the quote.
- Person block centered below separator:
  - Avatar: `w-12 h-12 rounded-full` with initials fallback (same pattern as `TeamSlide`).
  - Name: `font-display` 18px `text-slide-text`.
  - Role + Company: `font-body` 14px `text-muted`, separated by `·` if both present.
- Star rating (if provided): Lucide `Star` icons (`size={16}`) in `text-accent` for filled, `text-surface` for empty.

**Animation:** Quote fades in first (`opacity 0 → 1`, 600ms). Person block `fade-up` 300ms later.
