# Plan: New Template Library

## Background

The template library currently covers 19 slide layouts. This plan adds 15 new templates in three focused PRs, grouped by semantic role. Each PR is self-contained: it creates the template files, wires the smoke tests, and updates CLAUDE.md.

### Research notes (patterns every implementer must follow)

Before writing any code, read the templates listed below. They establish every convention used here.

**Full-bleed slides** (no `SlideLayout`, manage own padding):
- `TitleSlide` ‚Äî left accent bar + `TitleChrome` + `64px 80px` padding, `maxWidth: 920px`
- `SectionTitleSlide` ‚Äî same shell + `fitFontSize` + split-flap animation
- `TheEndSlide` ‚Äî centered flex, radial vignette, per-character `letter-rise` animation

**Content slides** (wrap with `SlideLayout`):
- `BulletSlide` ‚Äî minimal, shows canonical `SlideLayout + header` usage
- `CardSlide` ‚Äî staggered flip animation, hard-coded non-theme colors (justified by physical metaphor)
- `CompareSlide` ‚Äî shows animated row-by-row reveal

**Animation conventions:**
- One-shot stagger: `animationDelay = index * Nms`, applied via `style`
- CSS animations are defined in `tailwind.config.ts` (`keyframes` + `animation` map), referenced as `animate-*` class names
- New animations must be added to `tailwind.config.ts`; never use raw `@keyframes` in component files
- For export safety: if an animation loops, gate the loop on `isExportMode` from `../../src/utils/export`

**Color/token rules:**
- Content slides: always `bg-background`, `text-slide-text`, `text-muted`, `text-accent`, `bg-accent`, `bg-surface`, `border-surface`
- Hard-coded hex values are only acceptable when the visual metaphor demands it (playing card faces, photo placeholders) ‚Äî document the exception with a comment
- CSS variable form for non-Tailwind contexts (SVG fill, third-party props): `rgb(var(--color-accent))` etc.

**Props conventions:**
- Content slides: `header?: ReactNode` (rendered by `SlideLayout`)
- Full-bleed slides: no `header` prop ‚Äî direct string props only
- Array items are typed with named interfaces exported from the same file
- All props are typed; no `any`

**Files per template:**
1. `src/templates/[name]/[Name]Slide.tsx` ‚Äî main component + exported prop types
2. `src/templates/[name]/example.tsx` ‚Äî fully filled-in reference slide (never rendered by default)

**After each PR:**
- Add a smoke-test case per new template to `src/templates/smoke.test.tsx`
- Update the Template Catalog and Template Decision Guide table in `CLAUDE.md`
- Run `npm run build` ‚Äî fix all TypeScript errors before considering done
- Run `npm run export -- --deck=<any-deck> --format=png` (or create a throwaway demo deck) to visually validate

## Task 1 ‚Äî Structural / Framing Slides

Five templates that orient the audience: a visual section break, agenda, table of contents, closing slide, and pull quote.

### 1a. `SectionDividerSlide`

**When to use:** A full-bleed visual pause between major sections. Accepts a custom background color or image. Unlike `SectionTitleSlide` (which is text-focused with split-flap animation), this one is image/color-first ‚Äî the title floats over the visual.

**Shell:** Full-bleed (no `SlideLayout`). Pattern: `TitleSlide`/`SectionTitleSlide`.

```tsx
export interface SectionDividerSlideProps {
  title: string
  eyebrow?: string
  subtitle?: string
  /** CSS color string ‚Äî used as background when no backgroundImage is provided */
  backgroundColor?: string
  /** URL to a background image (fills the slide, dimmed by overlay) */
  backgroundImage?: string
  /** 0‚Äì1 overlay opacity to ensure text legibility over images. Default 0.55 */
  overlayOpacity?: number
}
```

**Visual design:**
- Root: `w-full h-full relative overflow-hidden flex`
- Background layer: `absolute inset-0` ‚Äî either `bg-background` (default), solid `backgroundColor`, or `<img>` / CSS `backgroundImage` with `object-fit: cover`
- Semi-transparent dark overlay (`rgba(0,0,0,overlayOpacity)`) over the image so text is always legible
- Left accent bar: `w-1 bg-accent flex-shrink-0`
- `TitleChrome` for dot-grid texture
- Content: `relative z-10 flex flex-col justify-center`, padding `64px 80px`, `maxWidth: 820px`
- Eyebrow: 12px mono, `text-accent`, `letterSpacing: '0.28em'`
- Title: large display font, `clamp(48px, 6vw, 80px)`, bold, `text-slide-text`
- Subtitle: `text-muted`, 20px body, appears with `animate-subtitle-rise`
- No animation on the title itself (static, unlike `SectionTitleSlide`) ‚Äî keeps it composable with any background

### 1b. `AgendaSlide`

**When to use:** Numbered list of topics at the start of a deck or section, with optional time estimates.

**Shell:** `SlideLayout` with `header?: ReactNode`.

```tsx
export interface AgendaItem {
  label: string
  /** Optional time estimate, e.g. "10 min" */
  time?: string
}

export interface AgendaSlideProps {
  header?: ReactNode
  items: AgendaItem[]
}
```

**Visual design:**
- Identical row structure to `BulletSlide`: `flex items-center gap-6 py-5 border-b border-surface first:border-t`
- Left column (fixed width ~48px): sequential number `01`, `02` ‚Ä¶ in `font-mono text-accent`, 13px, opacity 0.8
- Center column (`flex-1`): topic label in `font-body text-slide-text`, 21px
- Right column (optional, `flex-shrink-0`): time in `font-mono text-muted`, 13px, `letterSpacing: '0.08em'` ‚Äî only rendered when at least one item has a `time`
- No animation needed ‚Äî agenda is read, not revealed

- Title: `font-display font-bold text-slide-text`, 22px

### 1d. `ClosingSlide`

**When to use:** The final human-facing slide ‚Äî "thank you", call to action, and/or contact info. Mirrors `TitleSlide` energy. Distinct from `TheEndSlide` (which is purely decorative cinema-style).

**Shell:** Full-bleed (no `SlideLayout`). Pattern: `TitleSlide`.

```tsx
export interface ContactInfo {
  email?: string
  website?: string
  linkedIn?: string
}

export interface ClosingSlideProps {
  /** Main headline. Default: "Thank You" */
  headline?: string
  /** Secondary line below the headline */
  subtitle?: string
  /** Optional call-to-action phrase */
  cta?: string
  contact?: ContactInfo
}
```

**Visual design:**
- Same shell as `TitleSlide`: left accent bar + `TitleChrome` + content area `padding: 64px 80px`, `maxWidth: 920px`, vertically centered
- Headline: `font-display font-bold text-slide-text`, `clamp(72px, 8vw, 108px)`, with `animate-letter-rise` stagger on each character (reuse `TheEndSlide`'s per-character animation pattern)
- Accent separator line: `animate-line-expand` after headline settles
- Subtitle: `text-muted`, 22px, `animate-subtitle-rise`
- CTA: `text-accent font-mono`, 14px, `letterSpacing: '0.18em'`, uppercase ‚Äî appears as a small label above the headline
- Contact: rendered below the separator as an `inline-flex gap-8` row; each item is `font-mono text-muted`, 14px. Use Lucide icons (`Mail`, `Globe`, `Linkedin`) at `size={14}` inline with the text.

## Task 2 ‚Äî Data, Metrics & Process Slides

Five templates for displaying quantitative information, timelines, and structured processes.

### 2a. `BigNumberSlide`

**When to use:** A single KPI, statistic, or metric that deserves to fill the slide. Nothing else competes.

**Shell:** `SlideLayout` with `header?: ReactNode`.

```tsx
export type TrendDirection = 'up' | 'down' | 'neutral'

export interface BigNumberSlideProps {
  header?: ReactNode
  /** The number or value to display, e.g. "98%" or "$4.2M" or "12,400" */
  value: string
  /** Short label describing the metric, e.g. "Customer Satisfaction" */
  label: string
  /** Optional one-liner context, e.g. "Up from 91% in Q3" */
  context?: string
  /** Shows a trend arrow (‚Üë ‚Üì ‚Üí) colored accent/red/muted. Default: none */
  trend?: TrendDirection
}
```

**Visual design:**
- Content area: `flex-1 flex flex-col items-start justify-center gap-4`
- `value`: `font-display font-bold text-slide-text`, `clamp(100px, 13vw, 160px)`, `lineHeight: 0.9`, `letterSpacing: '-0.04em'`
  - Animate in: `animate-fade-up` single shot on mount (delay 0)
- Trend arrow (if provided): Lucide `TrendingUp` / `TrendingDown` / `Minus` icon, `size={48}`, positioned inline beside the value. Color: `text-accent` (up), `text-red-400` (down), `text-muted` (neutral). Note: `text-red-400` is a standard Tailwind color, acceptable here since it's a semantic signal color, not a design token override.
- `label`: `font-body text-muted`, 20px, `letterSpacing: '0.06em'`, separated from value by `24px mt`
- Accent underline: `w-16 h-0.5 bg-accent mt-4`
- `context` (optional): `font-body text-muted`, 16px, `fontStyle: 'italic'`

```tsx
  left: ColumnContent
  right: ColumnContent
  ratio?: '50/50' | '40/60' | '60/40'
}
```

**Visual design:**
- `flex-1 grid gap-8` with `grid-cols-2` (50/50) or `grid-cols-[2fr_3fr]` / `grid-cols-[3fr_2fr]`
- Each column: `flex flex-col gap-4`

- String list items: same row structure as `BulletSlide` ‚Äî number prefix, `font-body text-slide-text`, 17px, `border-b border-surface`

- Vertical divider between columns: `w-px bg-surface mx-2 self-stretch` (optional, only for 50/50)
