# Plan: Reduce Template Duplication

## Background

A thorough audit of all 19 template files in `src/templates/` uncovered two categories of egregious duplication worth fixing. Other patterns (animation timing constants, per-template transition strings) are either intentionally template-specific or too small to justify the abstraction overhead—those are left alone.

---

## Task 1 — Extract a shared `SlideLayout` wrapper component

**Problem.** Thirteen templates open with nearly identical boilerplate:

```tsx
<div className="w-full h-full bg-background flex flex-col overflow-hidden"
     style={{ padding: '60px 80px' }}>
  {header && <div className="flex-shrink-0 mb-10">{header}</div>}
  …
</div>
```

The `className` string is always identical. The padding and header margin drift across templates for no principled reason:

| Padding | Templates |
|---|---|
| `60px 80px` | BulletSlide, CardSlide, SplitFlapBulletSlide, GlossarySlide, TemperatureSlide |
| `52px 80px` | CompareSlide, EmbedSlide, HeatmapSlide, CycleSlide, PrismSlide, StackSlide |
| `52px 72px 48px` | ChartSlide, FlowSlide |

| Header margin | Templates |
|---|---|
| `mb-10` | BulletSlide, CardSlide, SplitFlapBulletSlide, GlossarySlide, TemperatureSlide |
| `mb-8`  | ChartSlide, StackSlide |
| `mb-6`  | EmbedSlide, FlowSlide, HeatmapSlide |
| `mb-5`  | CompareSlide |
| `mb-3`  | CycleSlide, PrismSlide |

**Solution.** Create `src/templates/common/SlideLayout.tsx` with a single canonical padding (`60px 80px`) and header margin (`mb-8`), and **update all 13 templates to use these identical values**. No `padding` or `headerMargin` props — the whole point is that every content slide looks the same.

```tsx
import type { ReactNode } from 'react'

interface SlideLayoutProps {
  children: ReactNode
  /** Optional header rendered above children with a consistent bottom margin */
  header?: ReactNode
  /** Any extra props to spread on the root div (e.g. onClick, role, style additions) */
  [key: string]: unknown
}

export function SlideLayout({ children, header, ...rest }: SlideLayoutProps): ReactNode {
  return (
    <div
      className="w-full h-full bg-background flex flex-col overflow-hidden"
      style={{ padding: '60px 80px' }}
      {...rest}
    >
      {header && <div className="flex-shrink-0 mb-8">{header}</div>}
      {children}
    </div>
  )
}
```

Every affected template drops its root `<div>` and `{header && …}` block in favour of `<SlideLayout header={header}>`. Templates that currently pass extra props to the root div (e.g. `GlossarySlide`'s `onClick`/`role`, `StackSlide`'s cursor `style`) pass them via rest props on `SlideLayout`.

After migrating all 13 templates, eyeball-check the result with `npm run export` and adjust content sizing inside any template that looks cramped or sparse at the new uniform spacing—without reintroducing per-template padding overrides.

**Excluded from this task** (their root divs are structurally different and should not be touched):
- `TitleSlide` — `relative overflow-hidden flex` + left accent bar, no header prop
- `SectionTitleSlide` — same structure as TitleSlide + split-flap chrome
- `TheEndSlide` — `flex items-center justify-center` (centered layout)
- `KeyTakeawaySlide` — uses `relative overflow-hidden` + absolute-positioned header
- `SplitSlide` — CSS grid root, no header, left/right slots
- `ImageSlide` — full-bleed image, no header

**Files changed:** `src/templates/common/SlideLayout.tsx` (new), plus the 13 template files listed in the padding table above.

**Also update `CLAUDE.md`** to replace the canonical template pattern snippet with one that uses `SlideLayout`.

## Task 2 — Centralize CSS color variable constants

**Problem.** Two templates independently define the same module-level constants for referencing CSS color variables in non-Tailwind contexts (Recharts fill props, React Flow node styles, SVG attributes):

`src/templates/chart/ChartSlide.tsx`:
```ts
const ACCENT  = 'rgb(var(--color-accent))'
const MUTED   = 'rgb(var(--color-muted))'
const SURFACE = 'rgb(var(--color-surface))'
const GRID    = 'rgb(var(--color-surface))'   // alias for surface
```

`src/templates/flow/FlowSlide.tsx`:
```ts
const ACCENT  = 'rgb(var(--color-accent))'
const MUTED   = 'rgb(var(--color-muted))'
const SURFACE = 'rgb(var(--color-surface))'
const BG      = 'rgb(var(--color-background))'
const TEXT    = 'rgb(var(--color-text))'
```

**Solution.** Add named exports to the existing `src/constants.ts` (which already holds `SLIDE_WIDTH` / `SLIDE_HEIGHT`):

```ts
// CSS variable references for use in inline styles, SVG, and third-party chart props.
// These resolve at runtime through the active theme.
export const COLOR_ACCENT     = 'rgb(var(--color-accent))'
export const COLOR_MUTED      = 'rgb(var(--color-muted))'
export const COLOR_SURFACE    = 'rgb(var(--color-surface))'
export const COLOR_BACKGROUND = 'rgb(var(--color-background))'
export const COLOR_TEXT       = 'rgb(var(--color-text))'
```

Then delete the local constant declarations in `ChartSlide.tsx` and `FlowSlide.tsx`, replacing each usage with an import from `../../constants`.

**Files changed:** `src/constants.ts`, `src/templates/chart/ChartSlide.tsx`, `src/templates/flow/FlowSlide.tsx`.
